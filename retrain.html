Guide de ré-entraînement (Google Colab)
1
Exporter les images classifiées manuellement
Téléchargez les images de chaque type depuis la bibliothèque. Les images sont stockées dans stored_documents/<nom_type>/. Créez une archive ZIP avec la structure suivante :
dataset/ ├── rib/ ← images existantes ├── cheque/ ├── acte_naissance/ ├── ... └── certificat_vie/ ← nouveau type (50+ images) ├── img001.jpg ├── img002.jpg └── ...
💡 Copiez simplement le dossier stored_documents/ en renommant les sous-dossiers si nécessaire.
2
Ouvrir le notebook Colab et uploader le dataset
Ouvrez votre notebook multimodale.ipynb dans Google Colab. Modifiez uniquement la cellule de configuration (CELLULE 4) pour ajouter le nouveau type :
CLASSES = [ "rib", "cheque", "acte_de_naissance", "tableau_amortissement", "assurance", "attestation_de_solde", "acte_heredite", "certificat_vie", # ← Nouveau type ajouté ici # Ajoutez d'autres nouveaux types ici ]
⚠ C'est le seul changement à faire dans le code. Toutes les autres cellules restent identiques.
3
Lancer l'entraînement complet
Exécutez toutes les cellules du notebook dans l'ordre. L'entraînement prend environ 20-30 minutes sur GPU Colab.
# Dans Colab, exécuter toutes les cellules : # Menu → Runtime → Run all (Ctrl+F9) # Vérifier que le GPU est disponible : import torch print(torch.cuda.is_available()) # Doit afficher True
💡 Activez le GPU dans Colab : Runtime → Change runtime type → GPU (T4).
4
Télécharger et remplacer le modèle
Après l'entraînement, la dernière cellule télécharge automatiquement bank_doc_classifier_multimodal.pth. Remplacez l'ancien fichier dans votre dossier model/ :
classification_vf/ └── model/ └── bank_doc_classifier_multimodal.pth ← Remplacer ce fichier
⚠ Arrêtez le serveur Flask (Ctrl+C) avant de remplacer le fichier, puis relancez avec python app.py.
5
Vérifier que le nouveau type est reconnu
Après redémarrage, vérifiez dans le statut que le nouveau type apparaît dans les classes :
# Vérification via l'API curl http://localhost:5000/status | python -m json.tool | grep classes # Le nouveau type doit apparaître dans la liste des classes # ex: "certificat_vie" doit être présent
💡 Vous pouvez aussi tester directement dans l'interface en uploadant une image du nouveau type.
