Guide de ré-entraînement — Google Colab
1
Exporter les images classifiées
Téléchargez les images depuis la bibliothèque. Elles sont stockées dans stored_documents/<nom_type>/. Créez une archive ZIP avec cette structure :
dataset/ ├── rib/ ├── cheque/ ├── acte_naissance/ ├── … └── certificat_vie/ ← nouveau type (50+ images) ├── img001.jpg └── …
2
Modifier la liste des classes dans le notebook
Ouvrez multimodale.ipynb dans Google Colab. Modifiez uniquement la cellule de configuration (CELLULE 4) :
CLASSES = [ "rib", "cheque", "acte_de_naissance", "tableau_amortissement", "assurance", "attestation_de_solde", "acte_heredite", "certificat_vie", # ← Nouveau type ]
C'est le seul changement à effectuer. Toutes les autres cellules restent identiques.
3
Lancer l'entraînement complet
Exécutez toutes les cellules (environ 20–30 min sur GPU Colab T4).
# Activer le GPU : Runtime → Change runtime type → T4 # Puis lancer tout : Runtime → Run all (Ctrl+F9) import torch print(torch.cuda.is_available()) # Doit afficher True
Activez le GPU gratuit dans Colab : Runtime → Change runtime type → GPU (T4).
4
Télécharger et remplacer le modèle
Après entraînement, la dernière cellule exporte bank_doc_classifier_multimodal.pth. Remplacez l'ancien :
classification_vf/ └── model/ └── bank_doc_classifier_multimodal.pth ← remplacer
Arrêtez Flask (Ctrl+C) avant de remplacer, puis relancez : python app.py
5
Vérifier que le nouveau type est reconnu
Après redémarrage, vérifiez que le type apparaît dans les classes reconnues :
curl http://localhost:5000/status | python -m json.tool | grep classes # "certificat_vie" doit apparaître dans la liste
Testez directement dans l'interface en uploadant une image du nouveau type.
